# Hayverse — нодовая среда открытой киновселенной

Память проекта по направлению **Hayverse**: нодовое UGC-приложение для сборки
сцен/фильмов/игр из объектов армянского мира. Микс «Weavy × Git»: пользователи
собирают сцены из нод-объектов, режиссёр канонизирует версии через
git-подобный процесс, мультивселенная хранит все форки.

> Это «точка во времени» (сессии июнь 2026). При расхождении сверяйтесь с
> актуальными прототипами в [`prototypes/`](../prototypes/).

---

## Концепция

Open Source IP — киновселенная с открытым исходным кодом, работающая как
UGC-платформа («кинематограф-геймдизайн 2.0»).

- **Нода = высокоуровневый объект:** персонаж, локация, ландшафт, здание, город,
  транспорт, стилист, мейкап, мебель, произведение искусства, одежда (каталог
  армянских дизайнеров), музыка, сценарий, раскадровка.
- **Сборка сцены:** выбираешь объекты из базы, переконфигурируешь параметрами,
  соединяешь ноды → на выходе сцена (персонажи в локации с нужной одеждой,
  мебелью, искусством). Дальше — творческий/рекламный ролик или клип.
- **Git поверх кино:** форк → pull-request → модерация → merge в канон (`main`).
  Непринятое не удаляется, а живёт в **мультивселенной**. Режиссёр выбирает,
  какая версия идёт в канон конкретного сценария.
- **Сущности-комьюнити:** потомки реальных комьюнити (`urvakan`, `rambalkoshe`,
  `moct`) и бизнес-компании. В их «памяти» — фото/видео-архивы, участвующие в
  генерациях и референсах.
- **Просмотр/игра:** в любой момент собрать и отрендерить текущую версию фильма
  по хронологии; во время просмотра менять параметры (спокойный/динамичный, без
  диалогов, один кадр, вид сверху/сбоку). На любом таймкоде — войти в игру с этой
  точки и вернуться обратно в фильм.
- **Монетизация:** product placement 2.0 — бренды объявляют баунти, комьюнити
  снимает споты в сеттинге вселенной; лучшее уходит в рекламу.

Главные «вызовы на берегу»: (1) права/лицензирование (публичная оферта при
регистрации, provenance на каждом артефакте); (2) стандартизация ассетов
(форматы/разрешения, валидатор на загрузке).

## Прототипы (визуальные, кликабельные)

В [`prototypes/`](../prototypes/) — три автономных HTML-файла
(без сборки, открываются двойным кликом; иконки — Tabler с CDN, тема светлая/тёмная):

| Файл | Что демонстрирует |
|---|---|
| `prototype-1-node-editor.html` | Нодовый редактор: граф объектов, инспектор узла (выбор из базы + параметры), переключатель Канон/Мультивселенная, очередь pull-request на канонизацию, таймлайн фильма. |
| `prototype-2-film-game-player.html` | Плеер «фильм ↔ игра»: живые регуляторы (темп, камера classic/top/side/one-take, палитра, диалоги) и вход в игру с любого таймкода с возвратом в фильм. |
| `index.html` | Сшитый экран (актуальный): граф слева → плеер справа, изменение любого узла мгновенно перерисовывает кадр. |

## План интеграции внешних AI (Higgsfield, Google AI Studio, OpenAI, Seedance, Runway)

> Конкретику по эндпоинтам/лимитам/ценам провайдеров нужно сверять по живой
> документации — помечено ⚠️.

### Что даёт каждый провайдер

- **OpenAI** — рассуждение/оркестрация (GPT-5, o-series), structured outputs
  (JSON-спека сцены), function calling, **embeddings** (RAG по лору/архивам),
  **Moderation** (гейт UGC), gpt-image-1 (генерация+inpaint), TTS/Whisper, Sora 2.
- **Google AI Studio (Gemini)** — мультимодальность (понимание референсов),
  **Nano Banana** (Gemini Image — диалоговое редактирование с сохранением
  личности персонажа), **Imagen 4**, **Veo 3.x** (видео с нативным звуком),
  Lyria (музыка), TTS.
- **Higgsfield** ⚠️ — кинопресеты движения камеры (орбита, crash-zoom, bullet-time),
  **Soul** (эстетичная картинка + Soul ID для лица), **Speak** (говорящий аватар/
  липсинк), Ads-режим, Recast.
- **Seedance** (ByteDance) ⚠️ — топовое video gen (text/image-to-video), мультишот,
  first/last-frame, 1080p; основной «рабочий» движок.
- **Runway** ⚠️ — **Gen-4 References** (консистентные персонажи между шотами),
  **Act-One** (перенос актёрской игры на персонажа), **Aleph** (in-context
  редактирование готового видео: сменить ракурс/свет/погоду/время года).

### MVP-набор вызовов

OpenAI (рассуждение + модерация + embeddings) → Gemini Nano Banana (ключевой кадр
с консистентным лицом/одеждой) → Seedance (кадр→видео) → Higgsfield/Runway
(камера-мув / консистентность). Остальное — фазы 2–4.

### Архитектура

Узлы графа **не знают о вендорах**. Нода объявляет *capability*, между ней и
провайдером — наш слой:

```
Узел сцены → Generation Gateway → Capability Router → Adapter(OpenAI|Gemini|Higgsfield|Seedance|Runway)
                   │                      │
            Job Queue + Cache        политика: цена/качество/доступность/лицензия
                   │
        Artifact Store (content-addressed) + Provenance
```

Опоры:
1. **Capability-абстракция + адаптеры** — интерфейсы `LLMReason`, `ImageGen`,
   `ImageEdit`, `VideoGen`, `VideoEdit`, `PerfTransfer`, `TTS`, `STT`, `Embed`,
   `Moderate`. Смена модели = новый адаптер, граф не трогаем.
2. **Capability Router** — выбор модели по политике (качество/цена/латентность/
   права/доступность), фоллбэк при сбоях; режиссёр может пиннить модель на ноде.
3. **Всё асинхронно (job-based)** — очередь, webhook/polling, статус в ноде,
   ретраи с бэкоффом.
4. **Контентная адресация + кэш** — ключ = хэш(вход + seed + модель + версия);
   одинаковый вход не перегенерируем (деньги + воспроизводимость по сиду).
5. **Версионирование как git поверх артефактов** — иммутабельные артефакты, DAG
   версий, `main` (канон) + форки (мультивселенная), PR → модерация → merge;
   provenance (модель, сид, промпт, стоимость, автор, лицензия).
6. **Reference/Identity-пиннинг** — на каждого персонажа пакет идентичности
   (Gen-4 References / Soul ID / embedding), общий якорь для всех генераторов.
7. **RAG над архивами комьюнити** — фото/видео → embeddings → retrieval в промпты.
8. **Модерация-гейт перед канонизацией** — OpenAI Moderation + Gemini safety +
   ручное ревью; без прохождения артефакт остаётся в мультивселенной.

### Дорожная карта

- **Фаза 0 — фундамент:** Gateway, адаптеры, очередь, Artifact Store, provenance, роутер.
- **Фаза 1 — MVP «статичная сцена»:** LLM-сценарий → entity-ноды → ключевой кадр
  (Nano Banana/Soul) → один шот image-to-video (Seedance) → сборка на таймлайне.
- **Фаза 2 — диалоги и звук:** TTS + говорящий аватар/липсинк (Speak/Act-One),
  Veo/Sora со звуком, субтитры (STT), музыка (Lyria/ElevenLabs).
- **Фаза 3 — редактирование и мультивселенная:** Runway Aleph (вид сверху/сбоку,
  смена погоды/времени на готовом видео), форки/PR/merge, diff-вьюер версий.
- **Фаза 4 — игра + экономика:** игровой режим, бренд-баунти, маркетплейс, оферта.

## Каталог нод

### A. Внутренние (разрабатываем сами — данные, логика, без внешних моделей)

- **Мир/git-слой:** `Universe`, `Branch/Canon (main)`, `Fork/Multiverse`,
  `Commit/Version`, `PullRequest`, `Merge`, `Diff`, `Tag/Release`.
- **Сущности-объекты:** `Character`, `Costume/Clothing`, `Stylist`, `Makeup`,
  `Location`, `Landscape`, `City`, `Building`, `Interior`, `Furniture`, `Prop`,
  `Artwork`, `Vehicle`, `CommunityMemory` (urvakan/rambalkoshe/moct), `Brand/Company`.
- **Нарратив/режиссура:** `Screenplay/Script`, `Beat`, `Scene`, `Shot/Storyboard`,
  `Timeline/Sequence`, `Dialogue`, `Camera`, `Lighting/TimeOfDay`, `Weather`,
  `Blocking/Mise-en-scène`, `MusicCue`.
- **Композиция/управление:** `SceneComposer`, `Reference/Moodboard`, `Style`,
  `PromptTemplate`, `Seed/Determinism`, `ReferencePin`.
- **Платформа/ops:** `Artifact`, `Job`, `ModelRouter`, `Provenance`, `Cost/Budget`,
  `License/Rights`, `Moderation`, `Bounty/Brief`, `User/Author`, `Role/Permission`.
- **Воспроизведение:** `FilmRenderer/Assembly`, `GameMode/Interactive`, `ViewerParams`.

### B. Внешние (обёртки над сторонними API)

| Нода (capability) | Что делает | Провайдер: основной → фоллбэк |
|---|---|---|
| `LLMReason` | Сценарий, JSON-спека сцены, оркестрация | OpenAI GPT-5 → Gemini |
| `Embed` | Векторизация лора/архивов (RAG) | OpenAI text-embedding-3 → Gemini |
| `Moderate` | Гейт UGC перед каноном | OpenAI Moderation + Gemini safety |
| `ImageGen` | Генерация кадра/ассета | Higgsfield Soul / Imagen 4 / gpt-image-1 |
| `ImageEdit` (identity-safe) | Сменить одежду/деталь, лицо то же | Gemini Nano Banana → gpt-image-1 → Recast |
| `Inpaint/Outpaint/Matte` | Маски, расширение кадра, вырез фона | gpt-image-1 / Imagen / Nano Banana |
| `CharacterReference` | Пин идентичности между шотами | Runway Gen-4 References / Soul ID |
| `Text2Video` | Видео из текста | Seedance → Veo 3 → Sora 2 → Runway Gen-4 |
| `Image2Video` | Кадр(ы)→движение, first/last-frame | Seedance → Runway Gen-4 → Veo |
| `CameraMotionVideo` | Кинопресеты движения камеры | Higgsfield (профильная фича) |
| `VideoEdit/Re-angle/Relight` | Сменить ракурс/свет/погоду на видео | Runway Aleph |
| `PerformanceTransfer` | Игра/мимика актёра → персонаж | Runway Act-One |
| `TalkingAvatar/Lipsync` | Говорящий персонаж по аудио+кадру | Higgsfield Speak → Act-One |
| `VideoWithAudio` | Видео с нативным звуком | Veo 3 / Sora 2 |
| `TTS` | Озвучка диалогов | OpenAI TTS → Gemini TTS |
| `STT/Subtitles` | Транскрипт, субтитры | OpenAI Whisper → Gemini |
| `Music` | Саундтрек/скор | Google Lyria / ElevenLabs ⚠️ |
| `SFX/Foley` | Шумы, звуковой дизайн | ElevenLabs ⚠️ |
| `Upscale/Interpolate/Restore` | Апскейл, доводка fps/качества | Topaz / model-native ⚠️ |

### C. Гибридные (наша логика + внешний вызов под капотом)

`SceneComposer`, `FilmRenderer`, `TalkingAvatar`, `GameMode` — внутренние ноды,
которые в момент рендера дергают внешние capability через Gateway.

## Риски (PM)

- Стоимость/латентность видео → кэш по хэшу, превью на дешёвой модели, финал на дорогой, бюджет-нода.
- Дрейф провайдеров → обязательная абстракция; ни одна нода не завязана на вендора.
- Консистентность между моделями → единый `ReferencePin`.
- Права/лицензии → provenance на каждом артефакте + публичная оферта.
- Безопасность UGC → модерация-гейт перед merge в канон (у Higgsfield исторически есть NSFW-риски).
- Стандартизация ассетов → фиксированные форматы/разрешения, валидатор на загрузке.
